~~~~ VPSYS 0.4 ~~~~
1. Удалены неиспользуемые функции.
2. Ввод целых чисел в двоичном и шестнадцатиричном форматах.
3. Справка по help.
4. Добавление нескольких элементов одного типа в строке(&имя a, b, ...)
5. Добавление информации о среднем времени работы элемента в справку.
6. Запрещено изменять публичные параметры элемента из скрипта.
7. Разрыв связей через FREE.
8. Удаление элемента через DEL.
9. В интерактивном режиме: load "файл" - загрузить и выполнить файл,
load dbg "файл" - выполнить пошагово.
10. Копирование элементов (&& b = a, c = d@gr1, ...).
11. Команда Run без параметров приводит к однократному выполнению всей цепи
элементов. Чтобы поменять начальное время (по-умолчанию - 0), нужно изменить
переменную %tau0.
12. Создавать цепочку перед запуском с помощью команды Chain больше не обязательно.
Команда осталась, чтобы при желании можно было проверить, будет ли выполнено
построение, а также узнать последовательность обработки с помощью "chain?".
13. Возможность создания копии графа - блок - который можно использовать в
схеме как обычный элемент. Выходами элемента автоматически становятся
неподключенные коннекторы входящих в него элементов.
14. Переменные могут быть не только числами, но также строками или массивами.
Добавлены ссылки.
15. Доступ к элементу блока через точку.
16. Элементы: интегратор, дифференциатор. Несколько мат. функций.

~~~~ VPSYS 0.5 ~~~~
1. Для переменных определены: del - удалить, free - очистить (удалить ссылку или
установить =0), %...? - список переменных (с доп. обозначениями const - константа,
(r) - ссылка).
2. Дописан вызов help для блока.
3. Реализована возможность использовать методы данных (выражения %var.todo(...)).
В частности, определены некоторые методы для массивов и операций с матрицами.
4. Поддержка рациональных чисел.
5. Индикация завершения программы при вводе пустой строке :) Небольшое изменение
приветствия.
~~~~ VPSYS 0.51 ~~~~
1. Улучшен формат вывода при печати. В режиме "калькулятора" числа печатаются в
относительно полном виде, рациональные числа представляются в виде дроби. Возможность
менять число знаков после запятой.
2. Исправлен баг с использованием рациональных чисел для расчётов.
